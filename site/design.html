How it works

From a real session to a scored task

Sessions are screened for public repos at a fixed commit, sanitized, and turned into a sandboxed task. The agent reads the first user message; the simulator supplies the rest.

→

Reproducible environments

Each task clones its upstream repo at an exact base commit inside Docker, with the coding-agent harness version pinned for byte-stable reruns.

→

Weighted completeness goals

Beyond pass/fail, each task decomposes the ask into core / secondary / polish goals with explicit weights, so a near-miss scores fairly.

→

Many agents & models

The lite70 suite runs multiple agent scaffolds across frontier models with replicates, scoring on an agentic judge rather than narrow tests alone.

Cohorts

Agent × model matrix

Every task in the lite70 suite is run across each agent scaffold paired with each frontier model, with replicates for variance.

Composition

What's in the suite

The lite70 set spans languages, task types, and difficulty — from one-line bug fixes to multi-file features.
